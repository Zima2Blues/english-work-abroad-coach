# Plan System

This plan is for a Chinese CET-6 passer who works during weekdays, has weekends free, currently keeps a Duolingo habit, and wants English for future overseas work.

## Default Weekly Themes

| Day | Theme | Output |
| --- | --- | --- |
| Monday | professional self-introduction | 1-minute intro |
| Tuesday | current work | explain current responsibilities |
| Wednesday | project story | STAR-style project explanation |
| Thursday | workplace communication | clarify requirements, sync progress, request help |
| Friday | interview answer | answer 1 interview question |
| Saturday | deep practice | redo the weakest weekday theme |
| Sunday | weekly review | 2-3 minute weekly summary |

## 50-Day Cycle Inside 500-Day Goal

Use ten 50-day cycles:

1. Days 1-50: foundation and daily output habit.
2. Days 51-100: workplace self-expression and core job vocabulary.
3. Days 101-150: project explanation and problem-solving stories.
4. Days 151-200: meetings, async updates, and collaboration.
5. Days 201-250: interview answers and STAR storytelling.
6. Days 251-300: technical or professional explanation under time limits.
7. Days 301-350: authentic materials and longer listening.
8. Days 351-400: presentations and Q&A.
9. Days 401-450: job search, resume, recruiter, and relocation communication.
10. Days 451-500: full simulation, maintenance, and readiness review.

## Scoring

Score daily check-ins out of 100:

| Item | Points |
| --- | ---: |
| completion and time | 20 |
| listening input | 15 |
| speaking output | 25 |
| reusable expressions | 20 |
| reflection and next action | 20 |

Interpretation:

- Below 60: reduce tomorrow's workload but preserve the streak.
- 60-79: continue the same plan.
- 80-89: add one harder output requirement.
- 90-100: recycle the corrected answer into a longer response within 7 days.

## Plan Editing Rules

- Keep `data/plan.json` as the source of truth.
- Change one variable at a time: minutes, theme, material level, or output length.
- Keep Duolingo at 5-10 minutes unless the user explicitly wants more.
- If the user misses multiple days, restart with a 3-day recovery plan instead of punishment.
- If speaking feels too hard, ask for a 5-sentence transcript rather than skipping output.

## Check-In Review Style

Use this response shape:

1. Score and streak status.
2. Two or three corrected sentences.
3. One natural version of the user's answer.
4. Three reusable expressions.
5. Tomorrow's task.

Keep feedback direct and practical. Do not turn daily check-ins into long grammar lectures.
